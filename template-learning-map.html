最重要

每次写作只做 4 步

1

先看题目线索

Task 1 看“写给谁”；Task 2 看题目的动词和问法。

2

选大题型

例如：正式信、观点题、双方观点题、问题解决题。

3

看小题型

例如：投诉、申请、原因+正负判断、问题+解决。

4

按段落回答

不要把所有想法混在一起。每个主体段只完成一个任务。

5

替换内容

保留框架句，只改 [主题] [原因] [例子]。

6

最后检查

Task 1：3 个 bullet points。Task 2：题目的每一个问题都回答。

判断地图

先判断：我应该用哪套？

Task 1：先看收信人

Task 1 的大题型不是“投诉”或“申请”，而是 书信语气。投诉、申请等属于小题型。

  • 公司、酒店、学校、政府、陌生人 → 正式信
  • 房东、老师、经理、同事、邻居 → 半正式信
  • 朋友、家人、亲密熟人 → 非正式信

Task 2：先看题目动词

Task 2 的大题型由题目最后一句决定。看到关键词就直接选。

  • Do you agree / good or bad → 观点 / 判断类
  • Discuss both views → 双方观点类
  • Advantages and disadvantages / outweigh → 优缺点类
  • Problems / causes / solutions → 原因、问题、解决类
  • 题目有两个不同问题 → 双问题 / 混合问法
TASK 1

3 个书信框架：先选语气，再选目的

Task 1 的核心不是写得很复杂，而是：称呼正确、语气正确、3 个 bullet points 全部写到。

大题型写给谁常见小题型结尾
正式信 Formal公司、酒店、学校、政府、陌生人投诉、咨询、申请、请求、反馈、正式邀请Yours faithfully
半正式信 Semi-formal房东、老师、经理、同事、邻居、旧老师请求、道歉+赔偿、感谢、邀请、安排、分享经历Yours sincerely
非正式信 Informal朋友、家人、亲密熟人邀请、道歉、建议、感谢、安排、借东西、分享Best wishes / Take care

01 · 正式信

Formal Letter · 180–200 words

题目线索：write to the manager / company / office / council

complaint 投诉enquiry 咨询application 申请request 请求feedback 反馈
Dear Sir or Madam, I am writing to [写信目的]. Firstly, I would like to explain [背景 / 问题]. In addition, [第二个 bullet point]. Finally, I would be grateful if you could [你的请求 / 解决办法]. Yours faithfully,
  • 投诉：问题 → 影响 → 要求怎么解决。
  • 咨询：你想参加什么 → 问 2–3 个信息。
  • 申请：申请什么 → 经验 → 可工作时间。

02 · 半正式信

Semi-formal Letter · 180–200 words

题目线索：your landlord / teacher / manager / neighbour

request 请求apology 道歉thanks 感谢invitation 邀请arrangement 安排
Dear Mr / Ms [姓], I hope you are well. I am writing about [事情]. The reason is that [原因 / 背景]. I would also like to explain [第二个 bullet point]. Could you please [请求 / 行动]? Yours sincerely,
  • 请求：为什么需要 → 具体请求 → 你会怎样配合。
  • 道歉：道歉 → 解释 → 赔偿 / 修复。
  • 邀请：活动 → 为什么希望对方来 → 时间地点。

03 · 非正式信

Informal Letter · 180–200 words

题目线索：write to a friend / family member / close friend

invitation 邀请apology 道歉advice 建议sharing 分享arrangement 安排
Dear [名字], How are you? I am writing to [目的]. First of all, [第一个 bullet point]. Also, [第二个 bullet point]. I hope you can [第三个 bullet point / 请求]. Best wishes,
  • 建议：直接给建议 + 原因 + 小例子。
  • 分享+邀请：介绍新东西 → 为什么喜欢 → 邀请对方。
  • 借东西：借什么 → 为什么 → 什么时候归还。
Task 1 套用口诀：先决定语气，再决定目的。题目有 3 个 bullet points，就用 3 个主体任务。不要为了“高级”把信写得像 Task 2。
TASK 2

5 个议论文框架：小题型决定两段主体写什么

Task 2 不需要每道题都背一篇新的作文。固定使用四段：引言、主体一、主体二、结论。

Introduction: It is often argued that [改写主题]. I [明确立场 / 说明文章方向]. Body 1: The main reason is that [第一个任务]. For example, [简单例子]. Body 2: Another important point is that [第二个任务]. This means that [结果]. Conclusion: In conclusion, [重申你的答案].

01 · 观点 / 判断类

Opinion / Judgement

题目线索：Do you agree or disagree? / Is this good or bad? / What's your opinion?

agree / disagreegood / badpositive / negativeopinion
引言：明确写 I agree / disagree / think it is positive. 主体一：支持你立场的理由 1。 主体二：支持你立场的理由 2。 结论：重复同一个立场。
  • 关键：一开始就选边，不要结尾才选。
  • 主体段：每段都支持你的立场，不要两边平均写。
  • 让步：可以写一句 Admittedly...，但不要变成第三个主体段。

02 · 双方观点类

Discuss both views + opinion

题目线索：Discuss both views and give your own opinion.

discussion + opinion
引言：说明有两种观点，并写自己的看法。 主体一：解释 View A 为什么有人支持。 主体二：解释 View B；写你为什么更支持这一边。 结论：明确你的最终看法。
  • 关键：两种观点都要写，不能只写自己。
  • 段落分配：On the one hand = A；On the other hand = B。
  • 个人观点：在引言和结论必须出现。

03 · 优缺点 / 权衡类

Advantages / Disadvantages / Outweigh

题目线索：What are the advantages and disadvantages? / Do advantages outweigh disadvantages?

advantages + disadvantagesoutweigh
普通优缺点：主体一写优点；主体二写缺点。 Outweigh：同样写优点和缺点，但引言与结论要说哪一边更大。
  • 普通题：可以中立，不必说哪一边更重要。
  • Outweigh：必须回答“谁更大”，不要只列清单。
  • 每段：一个主要优点 / 缺点 + 解释 + 例子就够。

04 · 原因 / 问题 / 解决类

Cause / Problem / Solution

题目线索：What problems does this cause? / What are the reasons? / What can be done?

problems + solutionsdamage + solutionsreasons + solutions
引言：说明这是一个问题。 主体一：只回答第一问：原因 / 问题 / 影响。 主体二：只回答第二问：解决办法。 结论：总结问题和可行解决办法。
  • 关键：题目问 problems，就不要主体一写 causes。
  • 解决办法：谁做 + 做什么 + 为什么有效。
  • 最常用：政府、学校、公司、个人可以做什么。

05 · 双问题 / 混合问法

Two-part / Mixed Questions

题目线索：题目连续问两个不同问题。例如 Why is this happening? Is it positive or negative?

reasons + opinionreasons + positive/negativereasons + advantages/disadvantagesprediction + reasonstwo judgement questionsopinion + extension
引言：直接说明这篇文章会回答两个问题。 主体一：只回答 Question 1。 主体二：只回答 Question 2。 结论：分别总结两个答案。
最容易错：“Why is this happening? Is it a positive or negative trend?” 不是普通观点题。主体一必须回答 Why，主体二才写 positive / negative。
  • 原因 + 正负：第一段原因，第二段判断正面或负面。
  • 原因 + 优缺点：第一段原因，第二段优点与缺点。
  • 两个判断：第一段回答第一个 Yes/No，第二段回答第二个 Yes/No。
学习方法

怎么学，才能真的在考场套出来？

第一阶段：先练“分类”，不要急着写全文

  • 第 1 天：拿 24 道 Task 1，只做两件事：写给谁？正式 / 半正式 / 非正式？
  • 第 2 天：给每道 Task 1 标一个小题型：投诉、咨询、申请、邀请、道歉等。
  • 第 3 天：拿 24 道 Task 2，只圈题目动词：agree、discuss、advantages、problems、why。
  • 目标：看到题目后 20 秒内能说出“大题型 + 小题型”。

第二阶段：先写提纲，再写全文

  • Task 1：写称呼 + 每个 bullet point 后面写 2 个细节。
  • Task 2：先写四行：立场 / 主体一 / 主体二 / 结论。
  • 不要：一看到题目就开始写第一句。这样很容易漏题。
  • 目标：提纲用 3 分钟，写作更稳。

第三阶段：一套模板，换 3 道题

  • 同一天只练一种大题型。
  • 例如今天练“观点题”：远程办公、校服、爱好重要性。
  • 框架不变，只换 [主题] [理由] [例子]。
  • 目标：明白模板是“框架”，不是死背内容。

第四阶段：开始限时

  • Task 1：先用 25 分钟，熟练后降到 20 分钟。
  • Task 2：先用 45 分钟，熟练后降到 40 分钟。
  • 写完后只检查：题目是否回答完、时态、单复数、拼写。
  • 目标：稳定完成，比追求难词更重要。
最适合你的背法：每天背 1 个大框架，不背 10 篇范文。先读框架 → 自己填 1 道题 → 换题再填 1 道 → 最后写 1 篇完整作文。
快速自测

看到这些题，你会选哪套？

题目 A：You stayed at a hotel and were unhappy with the service. Write to the manager.

大题型：Task 1 正式信。小题型：complaint / 投诉。结构：问题 → 影响 → 要求解决。

题目 B：Your friend will visit your city. Give advice about the best time to visit and things to do.

大题型：Task 1 非正式信。小题型：advice / 建议。结构：建议 → 原因 → 补充建议。

题目 C：Many people work from home. To what extent do you agree or disagree?

大题型：Task 2 观点 / 判断类。小题型：agree or disagree。主体一和主体二都支持你的立场。

题目 D：More people take photos at famous places. Why is this happening? Is it a positive or negative trend?

大题型：Task 2 双问题 / 混合问法。小题型：reasons + positive/negative。主体一回答 Why；主体二回答 positive or negative。

题目 E：Discuss both views and give your own opinion.

大题型：Task 2 双方观点类。小题型：discussion + opinion。主体一 A，主体二 B，开头和结尾写你的看法。
